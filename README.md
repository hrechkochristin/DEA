\# DEA Telemetry Analyzer



\*\*DEA Telemetry Analyzer\*\* — це вебзастосунок для автоматизованого парсингу, аналізу та 3D-візуалізації телеметричних даних з бінарних лог-файлів Ardupilot (`.bin`). 



Система автоматично витягує дані з датчиків GPS та IMU, розраховує ключові кінематичні показники (дистанцію, швидкості, прискорення) та генерує інтерактивну 3D-траєкторію польоту в локальній системі координат ENU з таймлайном для відстеження об'єкта в часі. Додатково інтегровано ШІ-асистента для швидкого формування технічного висновку про місію.



\---



\## 🛠 Обґрунтування технологічного стеку



Для розв'язання задачі ми обрали виключно Python-стек, що дозволило об'єднати складні математичні обчислення та інтерактивний інтерфейс без необхідності розгортати окремі сервери чи бази даних.



\* Інтерфейс (UI) — `Streamlit`: Найкращий вибір для хакатону. Дозволяє за лічені години побудувати повноцінний інтерактивний дашборд. Він ідеально працює з Python-бібліотеками для даних, автоматично керує станом (Session State) для повзунка часу і легко деплоїться у хмару без налаштування Nginx чи Docker.

\* Парсинг логів — `pymavlink`: Офіційний та найбільш надійний інструмент для роботи з MAVLink та бінарними файлами `.bin` від Ardupilot. Забезпечує швидке читання потоку даних без необхідності попередньої конвертації у `.log` чи `.mat`.

\* Обчислення (Math/Data) — `pandas` та `numpy`: Використані для векторизованих розрахунків. Замість повільних циклів ми застосовуємо методи `pandas` для інтегрування прискорень та обчислення дельт, що дозволяє миттєво обробляти сотні тисяч рядків телеметрії.

\* Конвертація координат — `pymap3d`: Легка та спеціалізована бібліотека для точного геодезичного перетворення глобальних координат (WGS-84) у локальну декартову систему (ENU), що є критичним для правильної 3D-візуалізації відносно точки старту.

\* Візуалізація — `plotly`: Обрано через її потужні можливості для побудови інтерактивних 3D-графіків (`Scatter3d`). Дозволяє користувачу крутити траєкторію, наближати її та динамічно розфарбовувати лінію градієнтом залежно від швидкості об'єкта.

\## Інструкція із запуску (Локально)



Проєкт не потребує розгортання важких контейнерів, достатньо стандартного віртуального середовища Python.



\### 1. Попередні вимоги

Переконайтеся, що на вашому комп'ютері встановлено \*\*Python 3.9\*\* або новішої версії.



\### 2. Клонування репозиторію

Склонуйте проєкт на свій комп'ютер та перейдіть у папку проєкту:

bash

git clone https://github.com/hrechkochristin/DEA

cd ВАШ\_РЕПОЗИТОРІЙ





\### 3. Створення віртуального середовища

Створіть та активуйте ізольоване середовище для залежностей:



\*\*Для Windows:\*\*

```bash

python -m venv venv

venv\\Scripts\\activate

```



\*\*Для macOS/Linux:\*\*

```bash

python3 -m venv venv

source venv/bin/activate

```



\### 4. Встановлення залежностей

Встановіть усі необхідні бібліотеки з файлу `requirements.txt`:

```bash

pip install -r requirements.txt

```

\*(Якщо файлу `requirements.txt` немає, виконайте: `pip install streamlit pandas numpy plotly pymap3d pymavlink google-generativeai python-dotenv`)\*



\### 5. Налаштування AI-асистента

Для роботи функції "Огляд від ШІ" створіть у корені проєкту файл `.env` та додайте туди ваш ключ API від Google Gemini:

```env

API\_KEY=ваш\_секретний\_ключ\_тут

```

\*(Примітка: Якщо ключа немає, додаток все одно працюватиме, але ШІ-звіт не згенерується).\*



\### 6. Запуск застосунку

Виконайте команду для запуску Streamlit-сервера:

```bash

streamlit run ui.py

```

\*(Якщо термінал не розпізнає команду, використовуйте `python -m streamlit run ui.py`)\*



Після цього ваш браузер автоматично відкриє дашборд за адресою `http://localhost:8501`. Завантажте файл `.bin` через меню зліва і натисніть "Аналізувати політ"!
